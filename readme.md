## ⚔️ 战役第一阶段：内存防爆与总线争夺战

**🎯 核心目标**：让极度消耗内存的 LwIP 协议栈与 OV5640 图像采集在 128KB 的 SRAM 中和平共处。

* **遇到问题**：
    * 加入 LwIP 后，以太网初始化失败（`lwip_comm_init` 返回 1）。
    * 图像解码频繁报错 `Prepare failed: 3`，串口疯狂吐出乱码 `fn?I?`。
* **尝试办法**：试图开启 DCMI FIFO，并将 JPEG 缓冲区移至外部 1MB SRAM ；试图将以太网 DMA 描述符也移至外部 SRAM。
* **真相与改善**：
    1.  **以太网的命门**：STM32F4 的以太网 MAC DMA **无法访问外部 SRAM (FSMC)**。底层收发缓冲必须放在内部 `SRAMIN`。
    2.  **DMA 泥头车效应**：DCMI 接收的 JPEG 数据如果大于缓冲区设定大小，会直接溢出踩碎相邻的 FreeRTOS 堆栈。
* **最终解决**：
    * **极致压榨与隔离**：回归“最快即最稳”的纯内部方案。关闭 DCMI FIFO 恢复单次传输。
    * 给 JPEG 划定绝对安全的 **32KB** 内部缓冲。
    * 将内部 `mymalloc` 管家备用金压缩至 **20KB**，专供以太网 DMA 使用。
    * 将 TJPGDec 解码器的 16KB `work` 数组扔进 CPU 独享的 **CCMRAM**。

## ⚔️ 战役第二阶段：物理管脚的“隐形刺客”

**🎯 核心目标**：打通所有外设，确保传感器和网络同时工作。

* **遇到问题**：代码逻辑完美，加入 LwIP 之前水泵能正常触发，加入之后自动模式死活不喷水，且系统表现出极度卡顿或“死机”状态。
* **深度排查**：我们一度怀疑是任务调度问题，直到敏锐地发现烟雾传感器使用的是 **PA1** 引脚。
* **真相与改善**：
    * **PA1 的不可侵犯性**：在 RMII 模式下，`PA1` 是网卡和单片机 MAC 层之间极其神圣的 **50MHz 基准时钟线**。
    * 烟雾传感器的 `ADC_Init` 将 PA1 强行改为了模拟输入，瞬间掐断了以太网的心脏，导致 MAC 层底层硬件无限报错，拖死了整个 CPU。
* **最终解决**：
    * **管脚搬家**：将烟雾传感器的线拔下，迁移至 `PA5`。
    * **精准替换**：修改代码为 `ADC1_Channel_5`，彻底解除了硬件级的短路冲突。

## ⚔️ 战役第三阶段：RTOS 的“饿死”与“核爆”

**🎯 核心目标**：让 AI 决策中枢能稳稳地发号施令，继电器不再“假死”。

* **遇到问题**：手动模式下水泵能转，自动模式下串口打印了视觉识别 `Fire: 1`，但水泵依然不转；或者串口突然吐出 `<?` 乱码后 ESP8266 彻底停止上报。
* **尝试办法**：加入继电器的防抖锁定（`fire_latch_counter`）；检查 `g_latest_fire_result` 全局变量是否被正确赋值。
* **真相与改善**：
    1.  **被绑架的 AI 大脑 (逻辑死穴)**：原本的 `report_task` 将“AI检测”与“ESP8266休眠5秒”绑在了一起。导致 AI 每 5 秒才能睁眼一次，水泵根本来不及响应。
    2.  **cJSON 的堆核爆 (乱码真凶)**：LwIP 初始化吃掉了大量 FreeRTOS 堆内存（Heap）。当任务调用 cJSON 组装浮点数时，堆内存枯竭，`malloc` 返回空指针，引发 HardFault 坠机，ESP8266 任务直接死亡。
* **最终解决**：
    * **快慢逻辑分离**：重构 `report_task`。AI 采集与决策设为 **快逻辑 (每100ms执行)**；cJSON 打包与透传设为 **慢逻辑 (计数器每5秒执行)**。水泵响应速度提升 50 倍。
    * **恢复生命线**：将 `FreeRTOSConfig.h` 中的 `configTOTAL_HEAP_SIZE` 调回充足的 **25KB**，并使用 `static` 修饰庞大的结构体，彻底解决了栈/堆溢出导致的坠机。

## 💡 核心感悟与经验提取

1.  **“底层决定上层建筑”**：在嵌入式开发中，代码跑飞未必是 C 语言逻辑写错了，极有可能是 DMA 踩了内存、堆栈溢出，或者是硬件引脚配置被覆盖。
2.  **局部变量的骗局**：在多任务和多模块协同中，一定要确保底层算法模块（如视觉处理）把关键结果塞进了**全局共享的结构体**中，否则决策中心只会拿到一堆默认值。
3.  **继电器的物理尊严**：代码里的 `On` 和 `Off` 之间如果只有几毫秒的间隔，机械继电器是绝对不会动作的。必须在软件层面加入 **Latch（锁存器）机制** 或使用 OS 原生延时，给它留足吸合时间。
4.  **敬畏 LwIP**：引入网络协议栈是对单片机资源的降维打击。必须重新审视系统的 Heap 分配、中断优先级和引脚复用状态。
