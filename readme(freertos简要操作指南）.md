📘 FreeRTOS 任务管理标准操作文档 (STM32F4版)
一、 核心概念速记
在 FreeRTOS 中，每一个功能（如按键检测、屏幕刷新、数据发送）都是一个独立的“任务 (Task)”。
•	任务就像是一个死循环，拥有自己专属的内存（任务栈）。
•	任务必须学会“休息”，不能一直霸占 CPU。当它休息（阻塞）时，系统会自动把 CPU 交给其他需要干活的任务。

二、 如何添加一个新任务？（标准 4 步法）
假设你现在要新增一个“LED 闪烁告警任务”，请严格按照以下步骤操作：
1. 定义任务句柄 (在文件顶部全局区域)
句柄就像是任务的“身份证号”，后续删除或挂起任务时都要用到它。
C
TaskHandle_t LedTask_Handler;
2. 声明任务函数 (在文件顶部)
C
void led_task(void *pvParameters);
3. 创建任务 (在 start_task 函数中)
使用 xTaskCreate 函数将任务挂载到系统中。
C
// 参数说明：(任务函数名, "任务代号", 栈大小(字), 参数, 优先级, &句柄)
xTaskCreate(led_task, "LedTask", 128, NULL, 2, &LedTask_Handler);
参数避坑指南：
•	栈大小：128 表示 128 个字（即 512 字节）。如果任务里有巨大的局部数组或调用了 printf，至少要给 512 甚至 1024。如果只是翻转 IO，64 或 128 就够了。
•	优先级：数字越大，优先级越高（通常 1~15）。紧急任务（如网络发送、急停）给高优先级，耗时且不紧急的任务（如图像处理）给低优先级。
4. 编写任务逻辑 (在文件末尾)
【铁律】：任务函数内部必须是一个死循环 while(1)，且循环内必须包含一个释放 CPU 的动作（如 vTaskDelay），绝对不能出现硬件死等！
C
void led_task(void *pvParameters)
{
    while(1)
    {
        // 你的业务逻辑
        LED0 = !LED0; 
        
        // 【关键】：延时 500 毫秒，期间交出 CPU 控制权
        vTaskDelay(500); 
    }
}

三、 如何移除/删除一个任务？
如果某个任务是一次性的（比如开机播放一段音乐），播放完就不需要了，你可以将其销毁以回收内存。
1. 任务自我销毁
在任务内部调用 vTaskDelete(NULL)。
C
void init_task(void *pvParameters)
{
    // 执行一次性初始化...
    printf("Init done!\r\n");
    
    // 销毁自己，把占用的 RAM 还给系统
    vTaskDelete(NULL); 
}
2. 猎杀其他任务
如果你想在“按键任务”中关掉“图像任务”，只需要传入它的句柄。
C
// 传入你要干掉的任务的“身份证号”
vTaskDelete(CameraTask_Handler);

四、 ⚠️ 保持系统不死的“三大军规”
1.	绝对禁止抢占优先级设置为 0~4 的中断调用 FreeRTOS API： 如果你在某个外设中断（比如 USART_IRQHandler）里调用了带有 FromISR 后缀的 FreeRTOS 函数，该中断的 PreemptionPriority 必须设置为 5~15。否则瞬间死机。
2.	警惕巨大的局部变量：
不要在任务内部写 char buf[2048];。如果实在需要，请加 static 改为全局变量，或者使用 pvPortMalloc() 动态申请，否则会触发 Stack Overflow（任务栈溢出）。
3.	不要用死等延时：
在任务中，永远用 vTaskDelay() 替代原来裸机中的 delay_ms()。
