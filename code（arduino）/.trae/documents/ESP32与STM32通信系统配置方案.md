# ESP32与STM32通信系统配置方案

## 一、硬件连接方案

### 1. 串口连接（主要通信通道）
- **ESP32 TX_PIN (1)** → **STM32 RX引脚** (如PA10)
- **ESP32 RX_PIN (3)** → **STM32 TX引脚** (如PA9)
- **GND** → **GND** (共地连接，必须连接)

### 2. 电源连接
- 建议使用独立电源为两个设备供电
- 若使用同一电源，确保电源容量足够
- 注意：ESP32使用3.3V逻辑电平，STM32通常也使用3.3V逻辑电平，可直接连接

## 二、STM32端配置方案

### 1. 串口初始化配置
- **串口选择**：使用USART1 (PA9/TX, PA10/RX) 或其他可用串口
- **波特率**：9600 bps (与ESP32保持一致)
- **数据格式**：8位数据位，无校验位，1位停止位 (8N1)
- **硬件流控制**：禁用

### 2. 数据接收与解析
- **接收缓冲区**：创建足够大的缓冲区接收完整数据帧
- **数据解析**：实现字符串解析函数，提取各传感器数据
- **数据格式**：解析"FLAME_AO:xxx,FLAME_DO:0/1,SMOKE_AO:xxx,FIRE:0/1,SMOKE_DET:0/1\n"格式

### 3. 代码结构
- 初始化串口
- 配置中断接收
- 实现数据解析函数
- 添加错误处理机制

## 三、时钟同步策略

### 1. 硬件时钟同步
- **使用外部晶振**：两个设备使用相同频率的外部晶振
- **主从模式**：可选择一个设备作为时钟源，另一设备作为从设备

### 2. 软件同步方法
- **时间戳同步**：在数据帧中添加时间戳
- **定期校准**：通过通信协议定期校准时钟
- **心跳机制**：实现心跳包确保通信连接稳定

### 3. 通信可靠性保障
- **校验和**：可在数据帧中添加校验和
- **重传机制**：实现简单的重传机制
- **超时处理**：添加接收超时处理

## 四、实施步骤

1. **硬件连接**：按照接线方案连接ESP32和STM32
2. **STM32初始化**：配置串口和相关引脚
3. **数据接收代码**：编写STM32端接收和解析代码
4. **测试通信**：验证数据传输是否正常
5. **优化调整**：根据实际测试结果调整参数

1. **硬件测试**：检查接线是否正确
2. **通信测试**：验证数据传输是否稳定
